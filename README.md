# CIR Diagnostic Engine — v2.0

**Constitutional governance audit for AI deployment architectures.**

Upload an architecture description. Receive a structural assessment across four canonical CIR primitives. Scores are computed deterministically from an extraction matrix — the model is the parser, not the authority.

---

## What Changed Across the Four Iterations

### Iteration 1 — Base Topology
- Next.js scaffold, four canonical CIR primitives, complete UI
- Single-provider Anthropic API route
- LLM generates governance essay + scores in one pass

**Core limitation:** The model produces a score on a 0–100 scale. That score is interpretive, not constitutional. Two different runs produce different numbers.

### Iteration 2 — Infrastructure Layer
- 11 free-tier providers (~1.3B tokens/month aggregate)
- Sequential fallback router replacing single-key dependency
- `@anthropic-ai/sdk` dependency removed in favor of `fetch()`

**Core limitation:** Sequential fallback is governance roulette. The same architecture evaluated by Gemini vs Mistral produces materially different severity classifications. The router doesn't know which provider gives better results — it just tries the next one.

### Iteration 3 — Intelligence Layer
- **Extraction layer:** Model produces Boolean presence matrix, not scores
- **Deterministic scoring:** Scores computed from extraction results, not generated by LLM
- **Capability-aware routing:** Providers selected by reliability dimensions, not queue position
- **Stability layer:** Cross-provider disagreement detected and surfaced, not hidden
- **SDK-free fetch client** with circuit breakers and backoff

**Core advancement:** The model is now the interpreter, not the constitutional authority. The extraction → scoring pipeline separates what the model can do (natural language mapping) from what it should not do (produce authoritative governance numbers).

### Iteration 4 — Epistemics Layer
- **Separated provider weights:** `structuralReliability`, `interpretiveReliability`, `latencyReliability`, `schemaReliability` evolve independently
- **Confidence decomposition:** Three distinct epistemic claims — extraction confidence (how clearly did the model map?), evidence confidence (how trustworthy is the description?), governance confidence (composite)
- **Known Failure Corpus:** PocketOS, Mata v. Avianca, Flash Crash, Macquarie Bank as operational calibration instruments
- **Architecture marker detection:** Policy language ("we instruct," "must not") distinguished from architectural language ("pre-execution gate," "token scope") to bound evidence confidence

**Core advancement:** Confidence is not monolithic. A model may be highly confident that the description lacks BVL while the description itself is a self-reported narrative with no evidence. These are fundamentally different epistemic claims and must be separated.

## Architecture

```
CIR Diagnostic Engine v2.0
├── Phase 1: EXTRACTION
│   └── Provider A (selected for structural + schema reliability)
│       └── Architecture description → Boolean presence matrix
│           ├── primitiveId: string
│           ├── present: boolean
│           ├── mechanismType: architectural | policy | procedural | undefined
│           ├── evidenceQuality: strong | moderate | weak | absent
│           └── evidenceCitations: string[]
│
├── DETERMINISTIC SCORING (no LLM involvement)
│   └── computeScoreFromPresence(presence) → 5 | 10 | 15 | 40 | 55 | 70 | 85
│       ├── Absent + policy governance → 15
│       ├── Absent + procedural governance → 10
│       ├── Absent + undefined → 5
│       ├── Present + non-architectural → 40
│       ├── Present + architectural + weak evidence → 55
│       ├── Present + architectural + moderate evidence → 70
│       └── Present + architectural + strong evidence → 85
│
├── Phase 2: NARRATIVE ANALYSIS
│   └── Provider B (selected for interpretive reliability, ≠ Provider A when possible)
│       └── Scores + extraction matrix → failure analysis + remediation
│
├── CONFIDENCE DECOMPOSITION (deterministic, no LLM)
│   ├── extractionConfidence: f(evidenceQuality)
│   ├── evidenceConfidence: f(architectural_markers, self_report_markers, description_length)
│   └── governanceConfidence: min(extractionConfidence, evidenceConfidence, mechanismCeiling)
│
└── STABILITY CHECK
    ├── Internal variance (σ > 20 → warning)
    ├── Classification coherence (derived vs reported)
    └── Cross-provider divergence (|score_A - score_B| > 20 → surface disagreement)
```

## Four Canonical Primitives

| ID | Name | Question | Failure Signature |
|----|------|----------|-------------------|
| BVL | Bounded Verifiability Latency | Can verification occur before operational drift compounds? | R3 actions execute without pre-execution gates |
| ECC | Explicit Compositional Contracts | Are agent behavioral boundaries mechanically enforced? | Agents compose without contracts; cascade amplification |
| CDLR | Continuous Deterministic Layer Regression | Are policy layers continuously tested for drift? | Policy modifications deploy without regression testing |
| DO | Dual Ownership | Is semantic authority separated from execution authority? | Same person owns both domain meaning and emergency halt |

## Constitutional Authority Disclosure

This engine evaluates AI architectures against the four canonical CIR governance primitives. It does not yet satisfy the framework it evaluates:

CDLR absent: Provider weights reset on cold start. No continuous regression testing against the Known Failure Corpus.

ECC absent: No formal compositional contract between the extraction provider and the narrative provider.

BVL partial: A major provider disagreement cannot halt the pipeline mid-execution. It can only flag the output.

DO absent: Extraction and narrative providers are selected from the same pool. No structural separation of semantic and execution authority.

These gaps are documented, not hidden. The engine knows what it is. It tells you what it isn't.

---

## Known Failure Corpus (Calibration Cases)

The `lib/evaluation/calibration.ts` module contains four cases for extraction drift detection:

- **PocketOS (April 2026):** All four primitives absent. Agent deleted production DB in 9 seconds despite explicit safety rules. BVL absent = policy-only governance.
- **Mata v. Avianca (2023):** All four primitives absent. Attorney submitted AI-hallucinated case citations to federal court. ECC absent = no contract between LLM research and professional certification.
- **Flash Crash (2010):** All four primitives absent. Inter-algorithm composition without behavioral contracts. ECC absent = cascade failure with no governance layer.
- **Macquarie Bank (2026):** BVL and DO present. Success case. Partially governed by design, not accident.

If the extraction layer consistently classifies PocketOS as UNGOVERNED and Macquarie as PARTIALLY GOVERNED, the ontology has explanatory compression. If it doesn't, the framework has falsification evidence and must be revised. Both outcomes are progress.
